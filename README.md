# WhistleCare Azure Pipeline Project

This repository contains the backend Express API for the WhistleCare application and the Azure CI/CD Pipeline configuration to containerize and deploy it.

## Infrastructure

- **Source Control**: GitHub
- **CI/CD**: Azure DevOps Pipelines
- **Container Registry**: Azure Container Registry (ACR)
- **Hosting**: Azure App Service (Web App for Containers)
- **Database**: MongoDB Atlas

## Local Development

Ensure you have Node.js and MongoDB installed, or use a MongoDB Atlas cluster.

1. Clone the repository.
2. Run `npm install` to install dependencies.
3. Create a `.env` file based on `.env.example` and add your `MONGO_URI`.
4. Run `npm start` to start the server locally on port 3000.

## API Endpoints

- `GET /health` : Health probe endpoint.
- `GET /api/reports` : Fetch all reports.
- `POST /api/reports` : Submit a new report.

## Pipeline Details

The `azure-pipelines.yml` file is configured to:
1. Build a Docker image based on the `Dockerfile`.
2. Push the image to an Azure Container Registry via a Service Principal connection.
3. Securely download the deployment Publish Profile generated by Azure App Service from the DevOps Library.
4. Deploy the newly built container to the Azure Web App.
